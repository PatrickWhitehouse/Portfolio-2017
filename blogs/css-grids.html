CSS Grids

Self initiated project

Aims and Objectives

The aim of this self-initiated project was to reproduce my current portfolio homepage by using CSS grids, an upcoming CSS property.

To achieve the aim, it was important that a good resource was found to learn CSS Grid, and also a suitable platform to test my code.

Definition

At the start of this project, I had no understanding of CSS Grids, however I had a good understanding of Flexbox and also had used frameworks such as Bootstrap in previous projects. Due to this, I decided it would be best to start from scratch and to learn the basics whilst documenting them.

As quite a lot, if not the majority of web traffic is from handheld devices, such as mobile phones and tablets, the web has had to adapt to suit these devices and fulfil their user’s needs. To help reduce development time and to speed up rapid prototyping, frameworks were created so that developers could create responsive, accessible and functional web applications with speed. Some examples of well-known frameworks are Bootstrap and Foundation by Zurb. Frameworks can be downloaded and included through a stylesheet or through a CDN (content delivery network). As frameworks tend to include quite a lot of default styles, these cause the stylesheets to be relatively large and could increase load times which will cause some developers to steer away from them. An alternative to frameworks is Flexbox. Flexbox was introduced in 2014 and is supported by most browsers. Flexbox gives developers and designers full control over their designs and most importantly, very little code is needed to create a complex layout. A relatively new CSS layout property is CSS Grids. Currently it is not supported by all browsers and must be enabled using flags in Chrome.

”Flexbox is essentially for laying out items in a single dimension – in a row OR a column. Grid is for layout of items in two dimensions – rows AND columns. “ (Andrew, 2016:online)

Location

How

As CSS Grids is only just emerging in modern web browsers, it was relatively hard to find tutorials and resources to learn about how it works. Luckily, I follow a well know key figure in the industry on Twitter, Rachel Andrew. Rachel had posted a link to a useful website which she had created, dedicated for beginners interested in learning CSS Grids and the syntax. The website is Grid by Example. I decided to document my learning on CodePen as it was easy to create an initial pen and to make various iterations of that pen as my learning developed.

First Iteration

For the first pen, a simple grid was created using six div elements within a wrapper which had the display:grid; property added to it. Within the wrapper, three columns and rows had been created. The code for the columns and rows were: 100px 100px 100px. This means that rows have a height of 100px and the columns have a width of 100px, which is repeated three times. As this grid uses a defined set width of the columns, this makes it unresponsive.

See the Pen Self initiated project - CSS Grids - 1 by Patrick Whitehouse (@patrickwhitehouse) on CodePen.

Second Iteraiton

As previously mentioned, using a defined width causes the grid to be unresponsive. A way around this is by using a new property called fraction units, written as ‘fr’ in grids. In this pen, the columns have been set to 1fr which means each column takes up one fraction unit of the available space, which is shared across three columns. If the first column was changed to ‘2fr’ this means it would take up two times as much space as the other two columns.

See the Pen Self initiated project - CSS Grids - 2 by Patrick Whitehouse (@patrickwhitehouse) on CodePen.

Third Iteration

What happens if you want the first column to get no bigger than 80px and no smaller than 50px? Simple. Add the ‘minmax’ property to the first column. The minmax property is self-explanatory. The first value you pass is the minimum width that column will resize to, the second property is the maximum width it’ll expand to.

See the Pen Self initiated project - CSS Grids - 3 by Patrick Whitehouse (@patrickwhitehouse) on CodePen.

Fourth Iteration

After the third iteration, I attempted to recreate the layout of my portfolios homepage. As my portfolio only uses a single column layout, it was relatively easy to achieve this in CSS grids.

See the Pen Self initiated project - CSS Grids - 4 by Patrick Whitehouse (@patrickwhitehouse) on CodePen.

Final Iteration

Once I understood CSS grids and how the children behaved inside of the grid container, I decided to recreate Manchester Metropolitan University’s homepage, using their images and colours. An issue I ran into during this iteration was remembering to group elements within divs for them to display in the same cell. CSS Grid tries to place each child into its own cell within the grid meaning, for example, if you had a H1 and P tag and wanted them in the same cell, you would have to group them in a div.

The final CSS Grid demonstrator.

Evaluation

It was relatively easy to recreate the homepage using CSS grids. This was down to grids allowing me to add other properties such as grid gap which meant usual CSS properties such as margin didn’t need to be added to space grid rows or columns. I believe CSS grids has an exciting future once all modern browsers decide to implement it. Sophisticated layouts can be created rather easily and elements can be moved around and positioned however the developer needs them with ease.
